Namespace curam.util.Dialog

Provides the ability to open arbitrary (non-UIM) content in a Curam dialog. It is required that the non-UIM page you are opening in a dialog resides on the same Internet domain as the Curam application you are integrating with.

To open a custom page in a dialog you have to

  • in the Curam application make the API call to load the custom page in a dialog
  • hook the custom page into the dialog API
To hook the custom page into the dialog API, you first need to load the code, preferably using dojo.require("curam.util.Dialog"); This assumes that you are using Dojo and you have access to the Curam infrastructure JavaScript libraries.

If this is not true for your pages, you will need to load the code in the page header with standard script HTML elements. The necesary script files are /CDEJ/jscript/curam/util/Dialog.js, /CDEJ/jscript/curam/dialog.js and /CDEJ/jscript/cdej.js.

The important API functions that must be called from your page are:

  • curam.util.Dialog.init in the page header.
  • curam.util.Dialog.pageLoadFinished when the page has been fully loaded.

To control the dialog size and title text you must register custom functions by calling the following API in the page header:

  • curam.util.Dialog.registerGetSizeFunc
  • curam.util.Dialog.registerGetTitleFunc

Optionally you can also register custom handlers for the following events:

  • AfterDisplay
  • BeforeClose
If linking from a dialog back into the Curam application is required, this is done using the curam.util.Dialog.close function.

close(refreshParent, newPageIdOrFullUrl, pageParameters)
Closes the dialog, optionally refreshing or redirecting the parent window. Must be called in the context of the page loaded in the dialog. I.e. you cannot close a dialog from an "outside" context.
Parameters:
{Boolean} refreshParent Optional, Default: false
Should the parent be refreshed when this dialog closes?
{String} newPageIdOrFullUrl Optional
ID of the page the parent window should be redirected to when this dialog closes. Alternatively a full URL including the page parameters can be passed.
{Object} pageParameters Optional
Page parameters to be used when redirecting the parent to the new page. The following format is expected: { param1Name:"value", param2Name:248 } The infrastructure handles URL-encoding the values so do NOT encode them yourself. If full URL is specified then the pageParameters are ignored.
closeAndSubmitParent(formParameters)
Closes the dialog and submit the parent page, where the form parameters will be optionally passed to. This method is design for the senario where the parent page has an ACTION phase. Must be called in the context of the page loaded in the dialog. I.e. you cannot close a dialog from an "outside" context.
Parameters:
{Object} formParameters Optional
Form parameters to be used in the form of the parent page when it is being submitted. The following format is expected: { param1Name:"value", param2Name:248 } Those parameters should not be encoded. The parameter name should be the order of the input fields in the form, e.g. '1' means the first input field in the form.
fileDownloadAnchorHandler(url)
Opens a modal window with confirm/cancel options for the current file download.

The modal informs the user that the file they are about to download contains sensitive personal information and warns them to clear their browser cache afterwards if using a public computer. It then lists instructions on how to clear the cache for their specific browser.

This method should be returned by the onclick function of a link when that link is being rendered.

Example Java Renderer code: 

anchor.setAttribute("onclick",
    "return curam.util.Dialog.fileDownloadAnchorHandler('" + url
        + "');");
The messages displayed to the user can be overridden using a copy of the GenericModalError.js.properties file.
Parameters:
{String} url
The URL to be accessed when the user confirms that they want to download the file.

Initializes the dialog infrastructure for an IEG script on a dialog. Must be called in the header of the page loaded in the dialog. This calls the onbeforeunload, because IE 11 and Edge browsers cannot handle the onunload call correctly. See RTC251740
open(path, pageParameters, dialogSize)
Opens the specified non-UIM page in a Curam dialog.
Parameters:
{String} path
URL path to the page to display in the dialog, without the query string.
{Object} pageParameters
An object containing the required page parameters, or null if no page parameters are required. The following format is expected: { param1Name:"value", param2Name:248 } The infrastructure handles URL-encoding the values so do NOT encode them yourself.
{Object} dialogSize Optional
An object representing the required size of the dialog in pixels. The following form is required: { width:500, height:300 } If size is not specified the default size will be used instead.
pageLoadFinished()
Notifies the dialog infrastructure that the page has been fully loaded. Must be called by the page after it finishes loading in the dialog. (E.g. dojo.addOnLoad(curam.Dialog.pageLoadFinished))
